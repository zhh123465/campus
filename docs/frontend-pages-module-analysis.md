## 1. 文档目标与分析范围

本文面向当前 `frontend/src` 下已实现的前端页面，系统梳理：

- 页面模块有哪些
- 每个模块在页面中的结构位置与信息分布
- 模块之间如何跳转、依赖哪些接口与状态
- 每个页面预期要达成的交互效果
- 当前实现与设计意图之间的差异

分析范围覆盖：

- 全局应用壳层
- 登录注册与访客入口
- 前台主业务页面
- 用户关系与个人展示页面
- 内容、空间、打卡、资源、搜索、消息、通知、积分、AI 页面
- 后台管理页面
- 404 页面

## 2. 前端整体结构概览

### 2.1 页面分层

当前前端可以分成四层：

1. **应用根层**
   - `App.vue`
   - 全局主题、消息、通知、路由出口

2. **路由与权限层**
   - `src/router/index.ts`
   - 负责游客页、登录态页面、管理员页面的进入控制

3. **布局层**
   - `src/layout/MainLayout.vue`：前台登录后主框架
   - `src/pages/admin/AdminLayout.vue`：后台主框架

4. **业务页面层**
   - `src/pages/*`
   - `src/pages/admin/*`

### 2.2 全局能力支撑

#### 2.2.1 全局请求能力

`src/api/request.ts` 提供统一请求封装：

- 基础前缀为 `/api/v1`
- 自动注入 `token`
- 自动注入 `X-Tenant-Id`
- 统一处理业务错误
- 401 时直接跳转 `/login`

这意味着所有业务模块都共享一套登录态与租户上下文机制。

#### 2.2.2 全局认证状态

`src/stores/auth.ts` 负责：

- 保存 `token`
- 保存当前用户 `user`
- 维护 `isLoggedIn`
- 登出时清理本地登录信息

#### 2.2.3 全局实时通知

`src/components/AppNotify.vue` + `src/composables/useWebSocket.ts` 提供：

- 页面挂载后建立 `/ws/notify` WebSocket 连接
- 接收到推送后弹出全局通知
- 断线 5 秒后自动重连

因此，通知中心不是唯一的通知入口；用户在任意页面停留时也会收到即时弹窗。

#### 2.2.4 富文本中的提及跳转

`src/components/MentionText.vue` 会解析文本中的 `@xxx`：

- 普通文本原样展示
- `@提及` 会变为可点击链接
- 点击后跳到 `/search?q=@xxx`

这使帖子、评论、用户内容与搜索模块形成弱耦合联动。

4. `requiresAdmin` 且角色不是 `TENANT_ADMIN` / `SUPER_ADMIN`：跳转 `/`

## 4. 全局布局模块分析

## 4.1 `App.vue`：应用根容器

### 模块位置

- 最外层：Naive UI 暗色主题配置
- 中层：消息与通知 Provider
- 内层：`AppNotify`
- 最内层：`router-view`

### 交互逻辑

- 页面切换都通过这里输出
- 全局消息、全局通知、主题在所有页面一致生效
- WebSocket 通知组件常驻，保证任何业务页都能收到通知弹窗

### 交互目标

- 统一视觉主题
- 统一消息反馈体验
- 让实时通知不依赖具体页面

---

## 4.2 `MainLayout.vue`：登录后前台主框架

### 模块位置

#### 左侧边栏

- Logo 区
- 主导航菜单

#### 顶部栏

- 搜索框
- 私信按钮
- 通知按钮
- 用户头像与昵称下拉

#### 主内容区

- `router-view`
- 页面切换带 `fade` 过渡动画

### 交互逻辑

1. **首次进入时补齐用户信息**
   - 若有 token 且 store 中 user 为空
   - 调 `getMe()` 拉当前用户资料
   - 失败则登出并跳回登录页

2. **搜索行为**
   - 顶部输入框回车后跳转 `/search?q=关键词`

3. **快捷入口行为**
   - 消息按钮跳 `/messages`
   - 通知按钮跳 `/notifications`

4. **用户下拉行为**
   - 个人中心跳 `/profile`
   - 退出登录调用 `logout()` 并清空登录态

### 交互目标

- 提供稳定统一的前台信息架构
- 让搜索、消息、通知、个人中心成为全局高频操作入口
- 减少页面切换时的认知成本

- 忘记密码入口

   - 忘记密码 → `/forgot-password`
   - 立即注册 → `/register`

### 要达成的交互效果

- 低摩擦完成登录

## 5.4 忘记密码页 `ForgotPassword.vue`

### 页面模块位置

- 标题卡片
- 两步进度条
- 第一步：输入注册邮箱
- 第二步：输入新密码、确认密码
- 返回登录链接

### 交互逻辑

1. 第一步获取重置令牌
   - 调 `forgotPassword(email)`
   - 返回 token 后保存到页面状态
   - 自动进入第二步

2. 第二步重置密码
   - 校验新密码与确认密码
   - 调 `resetPassword(email, token, newPassword)`
   - 成功后跳登录页

### 要达成的交互效果

- 把密码找回流程拆成可理解的步骤
- 让用户知道当前进度
- 支持从失败中回退重试

### 当前实现特点/问题

- 当前流程直接返回 token 到前端，更像开发态/简化态流程，不是典型邮件验证码体验

- 排序交互完整

## 6.2 发帖页 `PostCreate.vue`

### 页面模块位置

- 引用帖子提示区（可选）
- 帖子类型选择：普通 / 悬赏问答
- 标题输入
- 悬赏积分输入（仅 QA）
- 正文输入
- 话题输入与话题标签区
- 发布 / 取消按钮

### 交互逻辑

1. 引用帖子链路
   - 读取路由 query `quote`
   - 若存在则 `getPostById(quoteId)` 获取被引用帖子
   - 显示引用卡片，并支持取消引用

2. 帖子类型切换
   - `NORMAL`
   - `QA`
   - 选择 `QA` 后显示悬赏积分输入

3. 话题操作
   - 输入框回车或点击“添加”加入话题
   - 话题可删除
   - 自动避免重复话题

4. 提交逻辑
   - 正文必填
   - QA 帖子标题必填
   - 调 `createPost()`
   - 成功后跳转帖子详情页

5. 取消逻辑
   - `router.back()`

### 要达成的交互效果

- 支持普通表达与问答悬赏两种发帖心智
- 支持引用已有帖子形成讨论链
- 支持话题标签增强内容组织能力

## 6.3 帖子详情页 `PostDetail.vue`

### 页面模块位置

#### 帖子主体区

- 作者信息，可跳用户主页
- 精华标记
- 举报按钮
- 引用按钮
- 删除按钮（作者本人可见）
- 标题、正文、话题标签
- QA 信息区（悬赏积分、是否解决）
- 点赞 / 浏览 / 评论统计

#### 评论输入区

- 回复提示（若正在回复某条评论）
- 评论文本域
- 发表按钮

#### 评论列表区

每条评论包含：

- 评论者头像与昵称，可跳用户主页
- 评论时间
- 评论内容
- 点赞
- 回复
- 举报
- 采纳（仅 QA 且帖子作者且未解决）
- 删除（评论作者本人）

#### 举报弹窗

- 举报对象类型
- 举报原因
- 举报描述

### 交互逻辑

1. 页面初始化
   - `getPostById(id)` 获取帖子
   - `getComments(id, ..., qaSort)` 获取评论
   - 若帖子为 QA，再调 `getQaInfo(id)` 获取问答状态

2. 点赞帖子
   - 调 `toggleReaction(post.id, 'LIKE')`
   - 前端立即更新 `liked` 与 `likeCount`

3. 删除帖子
   - 作者本人可删
   - 删除成功返回广场页

4. 引用帖子
   - 点击“引用”跳 `/posts/new?quote=当前帖子ID`

5. 评论发布
   - 内容必填
   - 若是回复，则带上 `parentId/replyToId`
   - 提交后重新拉取评论并更新评论数

6. 评论点赞
   - 调 `toggleCommentReaction(comment.id, 'LIKE')`

7. 评论删除
   - 删除后重新拉评论列表并减少评论数

8. QA 采纳回答
   - 仅帖子作者可见
   - 调 `acceptAnswer(post.id, comment.id)`
   - 成功后更新 QA 状态为已解决

9. 举报链路
   - 支持举报帖子或评论
   - 调 `createReport()`

10. 用户跳转
   - 点作者、评论者均跳对应 `UserPage`

11. 文本提及联动
   - 正文、评论中的 `@用户名` 可跳搜索页

### 要达成的交互效果

- 成为帖子消费与互动的核心页面
- 同时承载：阅读、点赞、评论、回复、举报、引用、问答采纳
- 支撑普通帖子与问答帖两种内容形态

- 会话列表刷新依赖轮询，不是实时推送

- `peerName(msg)` 直接取 `msg.sender?.nickname`，当最后一条消息是我发出时，名称推断可能不稳定

## 17. 页面间的核心交互链路

下面按“用户旅程”梳理各模块间真正形成的主路径。

## 17.1 访客转化链路

1. 用户访问首页 `/`
2. 点击登录或注册
3. 注册成功跳登录
4. 登录成功跳广场 `/square`
5. 后续进入完整社区主流程

### 目标效果

- 完成从访客到用户的转化闭环

---

## 17.2 内容消费与互动链路

1. 用户进入广场 `/square`
2. 浏览帖子列表
3. 点击帖子进入详情 `/posts/:id`
4. 在详情页执行：
   - 点赞
   - 评论
   - 回复评论
   - 举报
   - 引用帖子
5. 若引用，则跳转到 `/posts/new?quote=id`
6. 发布后回到新帖子详情页

### 目标效果

- 形成内容生产—互动—再生产的循环

---

## 17.3 QA 问答链路

1. 用户在发帖页选择“悬赏问答”
2. 填写标题、内容、悬赏积分
3. 发布为 QA 帖子
4. 其他用户在帖子详情中评论作答
5. 帖子作者在评论区点击“采纳”
6. 帖子状态变为已解决

### 目标效果

- 让社区能支持知识问答场景，而不仅是普通动态

---

## 17.4 用户社交链路

1. 用户在帖子详情点击作者头像/昵称
2. 跳转到 `UserPage`
3. 在用户主页：
   - 关注/取消关注
   - 查看最近帖子
   - 查看成就
   - 发私信
4. 点击关注/粉丝进入 `FollowsList`
5. 再从列表进入其他用户主页

### 目标效果

- 让内容关系延伸为人际关系与持续关系

---

## 17.5 私信链路

1. 从用户主页点击私信
2. 跳 `/messages?peer=用户ID`
3. 自动打开对应聊天会话
4. 发送消息后，保存在消息流中
5. 会话列表轮询更新

### 目标效果

- 让公开互动延伸到私域交流

---

## 17.6 通知链路

1. 用户在站内产生被点赞、评论、回复、采纳、加入等事件
2. WebSocket 在全局弹出即时通知
3. 用户点击顶部通知入口进入 `/notifications`
4. 查看通知，点击后跳转到相关业务页面
5. 已读状态同步更新

### 目标效果

- 建立“事件发生 -> 即时提醒 -> 回流业务页”的召回链路

---

## 17.7 搜索链路

1. 用户通过全局搜索框或搜索页输入关键词
2. 进入 `/search`
3. 在结果中按类型筛选
4. 点击结果跳转到帖子/用户/资源/空间详情页

### 目标效果

- 让分散的业务实体能够被统一发现与检索

### 当前断点

- 顶部搜索与 `MentionText` 带 query 跳转到搜索页后，本页不会自动读取 `q` 并执行搜索

---

## 17.8 打卡挑战链路

1. 用户进入 `/checkin`
2. 浏览挑战列表
3. 进入某个挑战详情 `/checkin/:id`
4. 执行首次打卡或每日打卡
5. 查看排行榜和记录
6. 将自己的打卡记录分享到广场
7. 跳转形成新的帖子详情

### 目标效果

- 把“自律工具”变成“社区化成长玩法”

---

## 17.9 资源分享链路

1. 用户进入 `/resources`
2. 浏览资源列表
3. 进入详情页 `/resources/:id`
4. 下载资源
5. 或从列表页点击“上传资源”进入 `/resources/upload`
6. 上传成功后进入资源详情页

### 目标效果

- 建立校内资料流通与沉淀机制

---

## 17.10 后台治理链路

1. 管理员在广场中点击后台按钮或直接访问 `/admin`
2. 在后台左侧菜单切换管理模块
3. 对用户、帖子、空间、举报、敏感词、租户、AI 配置进行治理
4. 审计日志记录关键行为结果

### 目标效果

- 形成平台级运营、审核、治理与配置闭环

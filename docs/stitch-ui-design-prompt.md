## 2. 提示词正文

你现在是一名资深产品设计师 + 高级 UI/UX 设计师，请为一个名为 **CampusForum** 的项目设计一整套现代化、统一、可直接用于前端实现的 **Web UI 设计方案**。

### 一、项目背景

CampusForum 是一个面向高校场景的学习社区平台，核心定位是：

- 高校学习社群平台
- 多租户校园社区
- 社交互动 + 学习空间 + 打卡成长 + 资源分享 + AI 辅助
- 既有前台用户端，也有后台管理端

产品关键词：

- 校园感
- 学术感
- 科技感
- 年轻化
- 社群化
- 成长体系
- AI 增强
- 多模块统一体验

目标用户包括：

1. 普通学生用户
2. 校园社群运营者 / 空间创建者
3. 管理员 / 平台治理人员

### 二、设计目标

请你输出的 UI 设计必须满足以下目标：

1. **整体风格统一**
   - 前台和后台属于同一产品体系
   - 视觉语言统一，但层级清晰区分

2. **前台要有强烈的现代社区产品感**
   - 类似“新一代社区 + 学习平台 + 成长产品”的结合体
   - 不能做成传统教务系统风格
   - 不能老气、不能厚重、不能像传统论坛

3. **后台要有专业治理感**
   - 清晰、高效、可操作
   - 支持表格、筛选、批量操作、状态管理
   - 类似现代 SaaS 管理后台

4. **页面之间必须保持设计系统一致**
   - 色彩、圆角、阴影、卡片、按钮、输入框、标签、图标、空态、加载态要统一

5. **设计结果要可直接给前端落地**
   - 输出要考虑真实 Web 产品实现，而不是纯概念视觉海报

### 三、整体视觉风格要求

请采用以下视觉方向：

#### 1. 总体风格

- 深色科技风为主
- 带有校园学习社区气质
- 现代、轻盈、精致
- 有高级感但不过分炫技
- 可使用玻璃拟态 / 半透明卡片 / 柔和发光 / 渐变点缀

#### 2. 关键词

- Dark mode first
- Glassmorphism
- Neon accent
- Soft gradient
- Futuristic but usable
- Community + productivity hybrid

#### 3. 色彩建议

请使用一套适合该产品的完整色板，建议方向如下：

- 主色：靛蓝 / 蓝紫（如 Indigo、Violet）
- 辅助色：青蓝、紫色、橙色、金色
- 成功色：绿色
- 警告色：橙色
- 危险色：红色
- 背景：深色层次背景，不要纯黑一片
- 卡片：有明确层级区分

请设计时体现：

- 主背景
- 次背景
- 卡片背景
- 浮层背景
- 边框色
- 主文本色
- 次文本色
- 弱化文本色

#### 4. 组件风格建议

- 卡片圆角偏大（如 16~24）
- 按钮有主次层级
- 输入框简洁高级
- 标签有类型区分
- 表格不能生硬，要现代化
- 图标风格统一
- 空状态要美观，不只是简单文字
- 数据卡、趋势卡、资料卡都要具备明显层次感

  - 入驻高校数
  - 用户数
  - 帖子数
  - 部落数
  - 今日打卡数

  - 年轻化科技感

#### 1.2 登录页

- 学号（选填）
- 邮箱
- 密码
- 确认密码
- 注册按钮
- 去登录入口

#### 1.4 忘记密码页

设计成两步流程：

- 第一步：输入邮箱
- 第二步：输入新密码、确认密码
- 显示步骤进度

要求：

- 简洁、清晰、流程感强

# Stitch 分模块 UI设计提示词包

##1. 使用说明

这是一份给 Stitch 使用的 **分模块提示词包**，适合以下场景：

-你不想一次性把整个产品都丢给 Stitch
-你想分别生成首页、主站、空间、打卡、后台等模块
-你希望每个模块都能更聚焦、更稳定地产出高质量 UI

### 建议使用方式

1.先把下面的 **全局基础提示词** 发给 Stitch
2. 再追加某一个 **模块提示词**
3. 如果需要连续生成多个模块，保持同一设计上下文，逐个追加

### 推荐顺序

建议你按这个顺序生成：

1.设计系统与整体风格
2. 首页与账号体系
3. 登录后主框架 + 广场
4. 帖子详情 + 用户主页
5. 学习空间模块
6. 打卡模块
7.资源 + 搜索 + 通知 + 消息
8. AI 助手
9. 后台管理端

---

##2. 全局基础提示词

下面这段建议每次都先发给 Stitch，作为统一上下文。

### 全局基础提示词

你现在是一名资深产品设计师和高级 UI/UX设计师，请为一个名为 **CampusForum** 的 Web 产品设计高保真 UI。

这是一个面向高校场景的学习社区平台，产品定位是：

- 高校学习社群平台
- 多租户校园社区
- 社交互动 + 学习空间 + 打卡成长 +资源分享 + AI 辅助
- 同时包含前台用户端和后台管理端

请遵守以下全局设计要求：

#### 产品气质

- 年轻化
- 校园感
- 学术感
- 科技感
- 社区感
- 成长体系感
- AI 增强感

####视觉风格

- 深色科技风为主
-现代、精致、轻盈
-允许使用玻璃拟态、柔和发光、渐变点缀
-不能做成传统论坛
-不能做成传统教务系统
-不能做成老旧后台

#### 风格关键词

- Dark mode first
- Glassmorphism
- Modern SaaS
- Futuristic but usable
- Community + productivity hybrid

#### 配色建议

- 主色：Indigo / Violet
- 辅助色：Cyan / Purple / Orange / Gold
- 成功色：Green
- 警告色：Orange
- 危险色：Red
- 背景采用深色分层，不要一片纯黑

####组件要求

请统一设计系统，包括：

-颜色体系
- 字体层级
- 间距体系
- 圆角规范
- 阴影规范
- 卡片规范
- 按钮规范
- 输入框规范
- 标签规范
- 表格规范
- 空状态和加载状态规范

#### 最终输出要求

-以高保真 Web UI 为目标
- 页面要能直接支持真实前端实现
- 保持前台与后台属于同一套设计系统
- 在视觉上做到统一但层级分明

在此基础上，我会继续给你某一个具体模块，请只聚焦该模块输出高质量 UI方案。
